# Resource Loader API

The Resource Loader API enables programmatic creation and management of AB Tasty campaigns, variations, flags, and configurations. Use this to automate campaign setup from generated configurations.

## Overview

Resource Loader supports two platforms:

1. **Feature Experimentation (FE&R)**: Server-side campaigns, flags, goals, targeting keys
2. **Web Experimentation (WE)**: Client-side campaigns, variations, modifications

**Important:** Do NOT manually construct resource JSON. Always use the appropriate resource extractor prompt first:

- Feature Experimentation → Use [fear-resource-extractor.md](fear-resource-extractor.md)
- Web Experimentation → Use [we-resource-extractor.md](we-resource-extractor.md)

## Operations

### Load Feature Experimentation Resources

Creates/updates Feature Experimentation resources (campaigns, flags, goals, etc.).

**Required parameters:**

- `resourceLoaderContent` (object): JSON generated by `fear_resource_extractor` prompt
- `dryrun` (boolean): Simulate without creating resources

**Workflow:**

1. Use `fear_resource_extractor` prompt with campaign brief
2. Prompt generates proper JSON structure
3. Pass generated JSON to this operation
4. Resources created in AB Tasty platform

**Example workflow:**

```
User: "Create campaign 'Premium Feature Test' with flag 'premium-ui'
       for 50% of users on release v2.0.0"

Step 1: Use fear_resource_extractor prompt with this brief
Step 2: Prompt outputs valid JSON with resources array
Step 3: Pass JSON to resource_loader_api_load_featexp_resources
Step 4: Resources created
```

### Load Web Experimentation Resources

Creates/updates Web Experimentation resources (campaigns, variations, modifications).

**Required parameters:**

- `resourceLoaderContent` (object): JSON generated by `we_resource_extractor` prompt
- `dryrun` (boolean): Simulate without creating resources

**Workflow:**

1. Use `we_resource_extractor` prompt with campaign brief
2. Prompt generates proper JSON structure
3. Pass generated JSON to this operation
4. Resources created in AB Tasty platform

**Example workflow:**

```
User: "A/B test on homepage changing header text, 50/50 split"

Step 1: Use we_resource_extractor prompt with this brief
Step 2: Prompt outputs valid JSON with resources array
Step 3: Pass JSON to resource_loader_api_load_webexp_resources
Step 4: Campaign created
```

## Resource Structure

Both platforms use a standardized resource format with a `resources` array:

```json
{
  "needs_clarification": false,
  "questions": [],
  "resources": [
    {
      "type": "campaign|project|flag|goal|variation|modification",
      "$_ref": "c1",
      "action": "create|update|delete",
      "payload": {
        // Resource-specific fields
      }
    }
  ]
}
```

**Critical requirements:**

- Each resource MUST have: `type`, `$_ref`, `action`, `payload`
- Resources ordered by dependency (projects → campaigns → flags)
- Never use shorthand formats like `{"campaign": {...}}`

## Resource Types

### Feature Experimentation Resources

- **project**: Campaign containers/folders
- **campaign**: A/B tests or feature rollouts
- **flag**: Feature flags with types (boolean, string, number, json)
- **goal**: Analytics objectives (screenview, click, transaction, custom)
- **targeting-key**: Targeting criteria keys (string, number, boolean)

### Web Experimentation Resources

- **campaign**: A/B tests or personalizations
- **variation**: Test variations with traffic allocation
- **modification**: Element-level changes (text, HTML, CSS, JS)

## Dry Run Mode

Set `dryrun: true` to validate resource JSON without creating resources.

**Use dry run for:**

- Validating generated JSON structure
- Testing resource configurations
- Reviewing changes before applying

**Example:**

```json
{
  "resourceLoaderContent": {
    /* generated JSON */
  },
  "dryrun": true
}
```

Response confirms validation without creating resources.

## Clarification Handling

If resource extractor sets `needs_clarification: true`, the operation returns questions instead of creating resources.

**Response structure:**

```json
{
  "needs_clarification": true,
  "questions": [
    "What should the campaign be named?",
    "What are the allocation percentages?"
  ],
  "resources": [
    /* placeholder resources */
  ]
}
```

**Next steps:**

1. Answer clarification questions
2. Re-run resource extractor with additional details
3. Generated JSON will have `needs_clarification: false`
4. Load resources

## Credentials

### Feature Experimentation

Required credentials:

- `fear_rca_account_id`: Account ID
- `fear_rca_account_environment_id`: Environment ID
- `fear_rca_token`: RCA API token

### Web Experimentation

Required credentials:

- `we_account_id`: Account ID
- `we_token`: API token

**Security:** Store credentials as environment variables, never hardcode.

## Error Handling

**Invalid resource structure:**

- Error message indicates missing fields
- Use resource extractor to regenerate JSON

**Missing credentials:**

- Verify all required credentials provided
- Check credential validity in AB Tasty platform

**Validation errors:**

- Review error details in response
- Fix issues in campaign brief
- Regenerate with resource extractor

## Response Format

Successful load returns results for each resource:

```json
{
  "results": [
    {
      "type": "project",
      "$_ref": "p1",
      "status": "created",
      "id": "generated-id-123"
    },
    {
      "type": "campaign",
      "$_ref": "c1",
      "status": "created",
      "id": "campaign-id-456"
    }
  ]
}
```

## Best Practices

### Use Resource Extractors

- Always use prompts to generate JSON
- Never manually construct resource structures
- Prompts ensure schema compliance

### Validate First

- Use dry run mode before production
- Review generated JSON structure
- Verify targeting and allocations

### Provide Complete Briefs

- Include all campaign requirements
- Specify targeting criteria clearly
- Define variation splits upfront
- Reduces clarification rounds

### Handle Clarifications

- Answer all questions thoroughly
- Provide specific values, not ranges
- Re-run extractor with complete information

### Review Results

- Check returned IDs and statuses
- Verify resources in AB Tasty dashboard
- Test campaigns before activation
